# 韩语课程模板说明

> 🔴 **核心铁律**
>
> **铁律1：台词时间必须精确到帧**
> data-time-start / data-time-end 必须精准对应视频中的说话时间，误差不超过0.5秒
> - 台词不能提前亮，也不能延后灭
> - 相邻两句必须有间隙（如line1结束5.7s，line2开始5.9s）
> - 点击台词 → 视频跳到对应说话位置
> - 播放时 → 对应台词自动高亮
>
> **铁律2：所有代码修改一律交给 Claude Code**
> **严禁手动编辑 HTML/JS/CSS 文件！**
> - 任何对 index.html 的修改都必须 spawn Claude Code session 去执行
> - 自己只做需求确认和模板内容准备
> - 违反此条会导致结构错乱、标签漏闭合、课程互相嵌套等问题

## 四、每句台词 · 展开讲解6件套

| 字段 | 必须？ | 说明 | 示例 |
|:----|:---:|:-----|:----:|
| 📍**单词** | ✅ | 本条台词涉及的单词+词尾拆解 | 이름 — 名字(name) |
| 💬**短语** | ✅ | 用重点单词造的短搭配 | 연락처가 뭐예요? |
| 💬**造句** | ✅ | 用该句语法点的完整造句 | 이 책 제목이 뭐예요? |
| 📌**语法** | ✅ | 语法点讲解 + English | -이/가 뭐예요? 询问名称 |
| 💬**语法例句** | ✅ | 延伸例句（中英双语） | 내일이라고요? |
| 🎵**音变** | ✅ **必须加** | **音变div必须和ex平级，不能嵌套在语法例句里面** | 뭐예요→[뭐예요] |

**⚠️ 6件套缺一不可。短语和造句不能共用同一例句。**

**🔴 🎵音变的位置陷阱（第14课踩过的坑）：**
```html
<!-- ❌ 错误：音变嵌套在语法例句里面 -->
<div class="ex">💬 语法例句 <span class="c">...</span>
    <div class="phon">🎵 音变 ...</div>  <!-- 位置错误！ -->
</div>

<!-- ✅ 正确：音变和ex平级，都是line-exp的子级 -->
<div class="ex">💬 语法例句 <span class="c">...</span></div>  <!-- ex单独关闭 -->
<div class="phon"><b>🎵 音变</b> ...</div>  <!-- phon是兄弟元素，不是子级 -->
```

每条台词在 `.cn` 翻译下方增加 `.en` 英语翻译行：
```html
<p class="cn">→ 中文翻译</p>
<p class="en" style="font-size:12px;color:#888;margin:2px 0 0 16px">→ English translation</p>
```

## 五、LESSON_TIMES

```javascript
n:[
  {start:秒,end:秒},{start:秒,end:秒},...
]
```

**🔴 时间戳注意事项：**
1. **精确匹配视频实际说话时间**，从片源字幕中提取，误差≤0.5秒
2. **相邻两句要有0.2秒以上间隔**（如line1结束=5.7，line2开始=5.9），防止同时高亮
3. **data-time-start / data-time-end 与 LESSON_TIMES 必须完全一致**
4. **前10秒无对话时**，第一句从0秒开始高亮
5. 时间戳数组长度必须与 `lines` 数组完全一致

## 七、toggle 函数（添加在页面底部）

```html
<script>
function toggleVocab(el){
  var arrow=el.querySelector('span');
  var content=el.nextElementSibling;
  if(content.style.display==='none'){
    content.style.display='block';
    arrow.style.transform='rotate(90deg)';
  }else{
    content.style.display='none';
    arrow.style.transform='rotate(0deg)';
  }
}
function toggleGrammar(el){
  var arrow=el.querySelector('span');
  var content=el.nextElementSibling;
  if(content.style.display==='none'){
    content.style.display='block';
    arrow.style.transform='rotate(90deg)';
  }else{
    content.style.display='none';
    arrow.style.transform='rotate(0deg)';
  }
}
</script>
```

## 八、加课后检查清单（逐项打钩）

```
□ 1️⃣ 台词时间精确（从字幕提取，误差≤0.5秒）
□ 2️⃣ 相邻台词有0.2s+间隔（不重叠）
□ 3️⃣ 台词内容与视频完全一致（不用whisper，用片源字幕）
□ 4️⃣ 每句6件套完整：单词→短语→造句→语法→例句→🎵音变
□ 5️⃣ 🎵音变div 和 ex 平级（不嵌套在语法例句里）
□ 6️⃣ 每行的 <div> 数量 = </div> 数量（检查div平衡）
□ 7️⃣ handleSpeak 参数与韩语文本一致
□ 8️⃣ LESSONS.lines = LESSON_TIMES 数量一致
□ 9️⃣ data-time 属性 = LESSON_TIMES 一致
□ 🔟 单词汇总可折叠（toggleVocab），两列网格
□ 1️⃣1️⃣ 语法总结可折叠（toggleGrammar）
□ 1️⃣2️⃣ 底部顺序正确：小贴→单词→语法→已完成→笔记→口语
□ 1️⃣3️⃣ 已完成按钮只有一个（无重复）
□ 1️⃣4️⃣ 笔记区域有 textarea + noteStatus
□ 1️⃣5️⃣ 台词编号连续（1~n）
□ 1️⃣6️⃣ vocab 涵盖全部重点单词
□ 1️⃣7️⃣ 回归检查：其他课程页面没有被改坏（点开第1课、第13课验证）
□ 1️⃣8️⃣ 首页结构完整：没有缺`</div>`导致的卡片嵌套
□ 1️⃣9️⃣ 检查其他功能：测验模式、已完成、笔记、深色模式仍正常

### ⚠️ 回归检查范围

- **第1集（第1~9课）：不做任何检查，不动**
- **第2集已做完的课（第10~12课）：不做任何检查，不动**
- **只检查新增的课和首页**：主页能打开、新课卡片显示、点进去内容正常
- 已完成的课程如果没被修改，不会出问题，不需要重新验证
```

## 九、踩坑记录（每次犯过的错误，下次不再犯）

| # | 问题 | 原因 | 解决方法 |
|:---:|:---|:---|:---|
| 1 | Whisper 识别韩语不准 | whisper对韩语准确率低 | **必须用片源自带的字幕文件**，不用whisper |
| 2 | 时间戳凭感觉写，跟视频对不上 | 没有从字幕提取精确时间 | 从字幕SRT文件精确抄写start/end |
| 3 | 相邻台词时间重叠 | line1结束=6s，line2开始=6s | 留0.2s+间隔（如5.7→5.9） |
| 4 | 🎵音变漏了 | 忘记加第6件套 | 每句检查6件套完整性 |
| 5 | 🎵音变嵌套在语法例句里面 | 结构错误导致div不平衡 | 音变必须是ex的兄弟，不能是子级 |
| 6 | ✅已完成按钮出现两次 | 注入代码时重复添加 | 加课后检查只有一个done-toggle |
| 7 | 单词汇总一行堆砌 | 用分隔符连接所有单词 | 用两列网格，每条独立一行 |
| 8 | 语法总结漏了 | 忘记加底部语法框 | 底部必须加语法总结（可折叠） |
| 9 | 笔记状态漏了 | 忘记noteStatus | textarea后面加noteStatus span |
| 10 | 底部顺序不对 | 区块排列混乱 | 按 小贴→单词→语法→完成→笔记→口语 固定 |
| 11 | data-time和LESSON_TIMES不一致 | 只改了一处 | 两者必须完全一致 |
| 12 | 台词编号不连续 | 增删台词后忘记重新编号 | 增删后重新编号1~n |
| 13 | handleSpeak参数是旧文本 | 更新台词时忘记同步 | 更新台词同步更新handleSpeak() |
| 14 | div不平衡 | 新增行时忘记关闭div | 每行检查div和/div数量是否相等 |
| 15 | **手动改HTML（违反铁律）** | 觉得改一下就行，结果每次都出问题 | **严禁手动编辑HTML/JS/CSS，代码修改必须 spawn Claude Code 执行** |
| 16 | 做完没按检查清单打钩 | 改完直接说"好了" | 改完先逐项跑检查清单，全部通过再通知用户 |
| 17 | HTML标签漏闭合 | 卡片/页面缺少/div | 注入后检查所有div是否配对闭合 |
| 18 | 重复HTML标签 | 修复时加重复了 | 修复前看清楚上下文，改完再检查一遍 |
| 19 | 缺少div开头标签 | 复制粘贴漏了开头 | 注入的HTML从头到尾检查完整 |
| 20 | 多个Claude Code同时跑 | 同时发多个任务修改同一个文件 | 一次只跑一个Claude Code，等它完再跑下一个 |
| 21 | 子任务太复杂 | 一个任务包含太多步骤 | 拆成多个小任务，每个任务只做一件事 |
| 22 | 改完新课后没检查旧课 | 只关注新课，忽略了回归 | 改完必须检查其他课程是否正常（11-14课） |
| 23 | HDR视频转SDR没做色调映射 | 直接用ffmpeg转码，画面黢黑 | 必须加zscale滤镜做HDR→SDR色调映射 |
| 24 | 上传COS后没验证 | 以为上传成功了，实际没生效 | 上传后用curl验证HTTP 200 |
| 25 | 视频质量没检查就直接上线 | 没看画面效果就告诉用户好了 | 转码后先本地播放确认亮度/颜色正常，再上传 |
| 26 | 🔴 **HDR视频转SDR黢黑** | 直接用ffmpeg转码，没做色调映射 | **必须加zscale滤镜**做HDR→SDR色调映射：`zscale=t=linear:npl=100,format=gbrpf32le,zscale=p=bt709` |
