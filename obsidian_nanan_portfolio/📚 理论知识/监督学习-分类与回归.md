# 📊 监督学习 · 分类与回归

> 监督学习 = 有标签数据 → 找规律 → 预测
> 两大分支：分类（离散类别） + 回归（连续数值）

## 核心区别

| 维度 | 回归 | 分类 |
|:----|:----|:-----|
| 输出 | 连续数值 | 离散类别标签 |
| 本质 | 预测"多少" | 判断"是哪一类" |
| 例子 | 预测播放量、房价、温度 | 是不是垃圾邮件、好评/差评 |
| 类比 | 这菜多少钱？ | 这是川菜还是粤菜？ |

## 线性回归

- 在多个特征上做线性拟合
- 输入：点赞数、评论数、发布时间等
- 输出：一个数值（如下一条视频播放量预测）
- 本质：找一条线/一个平面，让所有数据点到它的距离最小

## 逻辑回归（名字带回归，本质是分类）

| 步骤 | 说明 |
|:----|:------|
| ① 底层计算 | 先算出连续数值（和线性回归一样） |
| ② Sigmoid函数 | 把连续值压缩到0~1之间，变成概率 |
| ③ 阈值判断 | 设定阈值（如0.5），≥为A类，<为B类 |

**核心：身体是回归的（算连续值），灵魂是分类的（按概率归类）**

### 实操场景（韩语自媒体）
- **线性回归**：根据点赞数、评论数、发布时间 → 预测下条视频播放量
- **逻辑回归**：根据同样因素 → 判断能否破万（能/不能 = 二元分类）

## 二元分类 vs 多元分类

| 类型 | 选项数 | 举例 |
|:----|:------|:-----|
| 二元分类 | 2个 | 垃圾短信/非垃圾短信 |
| 多元分类 | 3个及以上 | 山峰/大海/森林/沙漠 |

两者就一个区别：选项数量。

## KNN（K近邻）

> **KNN：新数据与已知数据逐一计算距离，取最近的K个样本，按邻居投票的多数类别作为预测结果。**

| 要点 | 说明 |
|:----|:------|
| 分类/回归 | 分类 |
| 距离计算 | 欧氏距离（两点间直线距离） |
| K值 | 无绝对标准，具体数据具体调 |
| K小 | 边界曲折，易过拟合 |
| K大 | 边界平滑，易欠拟合 |
| 致命缺点 | 每次预测都要与全部数据算距离，数据量大则慢 |
| 适用场景 | 数据量小、特征少 |

## 用户自述理解（2026.7.16）

> "回归它是从大量的输入的数据当中去预测一个数值，而分类是从有限的数据当中去判断分类。"
> "逻辑回归底层用连续数值预测，但通过Sigmoid函数变为0~1之间，按阈值分类。"
> "线性回归就是在多个数值上进行线性拟合来预测一个数值。"
