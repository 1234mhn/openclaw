# 🤖 KNN（K近邻算法）

## 一句话

> **新数据与已知数据逐一计算距离，取最近的K个样本，按邻居投票的多数类别作为预测结果。**

## 算法流程

| 步骤 | 说明 |
|:---|:------|
| ① 算距离 | 新数据点与每个已知数据点计算欧氏距离 |
| ② 排序 | 按距离从小到大排序（argsort） |
| ③ 取前K个 | 取距离最近的K个作为"邻居" |
| ④ 投票 | K个邻居各自所属类别投票，票数最多的为预测类别 |

## K值选择

| K值 | 问题 |
|:---|:-----|
| K太小（如K=1） | 易受离群点影响，过拟合 |
| K太大（如K=全部） | 永远输出最多的类，欠拟合 |
| K取奇数 | 避免平票（如K=3,5,7） |
| 最佳K值 | 通过交叉验证确定 |

### K值的本质

> K=3和K=5可能结果完全不同：最近3个是(A,A,B)→A胜；最近5个是(A,A,B,B,B)→B胜

## 鸢尾花数据集实战

### 完整流程（共7步）

| 步骤 | 代码 |
|:---|:------|
| ① 加载数据 | `load_iris()` |
| ② 切分数据 | `train_test_split(X, y, test_size=0.3)` |
| ③ 创建分类器 | `KNeighborsClassifier(n_neighbors=3)` |
| ④ 训练 | `clf.fit(X_train, y_train)` |
| ⑤ 预测 | `clf.predict(X_test)` |
| ⑥ 评估 | `accuracy_score(y_test, y_pred)` |
| ⑦ 调参 | 改变K值，重复③~⑥ |

### 结果
- K=3时，测试集准确率：**100%** ✅
- 鸢尾花数据集特征分离度高，KNN轻松完美分类

## KNN的优缺点

| 优点 | 缺点 |
|:---|:------|
| 简单直观 | 数据量大时极慢（逐点算距离） |
| 无需训练（惰性学习） | 高维数据效果差（维度灾难） |
| 可解释性强 | 对不平衡数据有偏好 |
| 适合小数据集 | 对特征尺度敏感（需归一化） |

## 进阶话题

| 话题 | 说明 |
|:---|:------|
| 距离度量 | 不止欧氏距离，还有曼哈顿、闵可夫斯基、余弦 |
| 归一化 | Min-Max / Z-score，让各特征公平投票 |
| KNN做回归 | 找到最近K个邻居，取平均值作为预测值 |
| 加权KNN | 近的邻居投票权重大，远的权重小 |
| 交叉验证 | 最科学选K值的方法 |
