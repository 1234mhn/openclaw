## 2. 🎬 韩语课程更新SOP

> 触发词：「加课」「更新新课」「第X课」
> 页面地址：https://nannan2026.cn/korean/

### 一句话流程

```
你说「加第X课」
→ 楠楠回复"收到"
→ 派🎓课程产品师
→ SOP执行全流程
→ 验收OK通知用户
中间不来回改
```

### 用户需要提供的
```
课号：第几课
视频开始时间：
视频结束时间：
```
台词不用发，从片源字幕提取。

### 执行流程

#### ① 视频处理（🎓课程产品师 → 🤖 Claude Code）
- 用COS签名URL直接截取（不下载原片！）
- ffmpeg HDR→SDR转码
- 上传截好的视频回COS

#### ② 提取台词（优先本地MKV + mkvextract）
- 必须用片源字幕提取（不用Whisper！）
- 从MKV提取韩语字幕+中文字幕
- 筛选目标时间范围台词

#### ③ 填充页面模板
按模板规范依次填充：
- 首页课程卡片
- LESSONS数据（课名/视频链接/单词/台词）
- 每句6件套：单词→短语→造句→语法→例句→🎵音变
- 语法总结（可折叠）
- 单词汇总（两列网格）
- 已完成按钮 + 笔记区 + 口语练习

#### ④ 自检清单
```
□ 台词时间精确（误差≤0.5秒，相邻留0.2s间隔）
□ 台词来自字幕提取，内容与视频一致
□ 每句6件套完整
□ handleSpeak参数与台词文本一致
□ LESSONS.lines 与 LESSON_TIMES 数量一致
□ 底部顺序正确：小贴→单词→语法→完成→笔记→口语
□ 单词汇总两列网格
□ 语法总结可折叠
□ 已完成按钮只有一个
```

#### ⑤ 备份 + 注入 + 回归检查
```bash
cp index.html index_backup_YYYYMMDD.html
```
- ✅ 检查第1课和第13课没被改坏
- ✅ 首页课程卡片完整
- ✅ 视频能播放
- ✅ 全部通过才通知用户

### 踩坑记录

| # | 问题 | 解法 |
|:-:|:----|:----|
| 1 | Whisper识别不准 | 必须用片源字幕 |
| 2 | 时间戳手写对不上 | 从字幕精确抄写 |
| 3 | 🎵音变漏了 | 每句6件套第6个 |
| 4 | 语法总结漏了 | 底部必须加（可折叠） |
| 5 | 单词一行堆砌 | 两列网格，每条一行 |
| 6 | 底部顺序不对 | 小贴→单词→语法→完成→笔记→口语 |
| 7 | 已完成按钮出现两次 | 检查是否只有一个 |
| 8 | 不小心下载了原片 | 🔴 绝对不下载COS原片到本地 |

### 做课时铁律（2026.7.6重申）

1. 台词数量 = LESSONS.lines = LESSON_TIMES 三者完全一致，一一对应
2. 台词翻译逐句核对，不能错翻漏翻
3. 每句6件套完整（单词/短语/造句/语法/例句/音变），缺一不可
4. 时间戳从字幕精确抄写，误差≤0.5秒，相邻留0.2s间隔
5. 注入后回归检查第1课和第13课，确认没被改坏
6. 全部通过再通知用户，不给看半成品
7. 用户说文件在COS上 → 先信用户，多换路径查
8. **先生成本课标题**：根据台词提炼标题（简短表意，如「介绍朋友」）

## 7. 🇰🇷🇺🇸 语言复利笔记SOP（场景学习卡）

> 详细需求文档：`项目/韩语场景学习卡_需求文档.md`

### 一句话描述
构建可扩展的场景化语言复利笔记。**主页展示所有场景 → 选择一个场景 → 进入手绘卡通场景 → 点击物品学单词**。英语和韩语共用同一套交互模板，只需替换场景数据和插画即可无限扩展。

### 技术栈
- **前端**：纯 HTML + CSS + JavaScript（单文件模式）
- **场景图**：AI 绘图工具生成手绘卡通风格 PNG/WebP
- **发音**：edge-tts 预生成 MP3，base64 内嵌到 HTML
- **数据驱动**：所有场景数据定义在 JS 对象中，新增场景只需加数据 + 图片 + 音频

### 数据模板结构

```javascript
const SCENES = {
  living: {       // 场景 ID
    name: "거실",       // 韩文/英文名称
    nameCn: "客厅",     // 中文
    emoji: "🛋️",
    image: "living_room.webp",
    items: [
      { 
        ko: "소파",              // 目标语言
        pron: "소·파",           // 音节拆解
        cn: "沙发",              // 中文
        zone: { left: "4%", top: "54%", width: "45%", height: "24%" },
        label: { left: "3%", top: "60%" },
        connector: { x1: 18, y1: 370, x2: 100, y2: 415 }
      },
    ]
  },
  // ... 更多场景
}
```

### 交互逻辑

| 操作 | 效果 |
|:----|:-----|
| 点击物品 | 区域高亮→标签高亮→播放发音→弹窗（韩文+中文）→1秒后消失 |
| 全部朗读 | 按顺序遍历→逐个高亮+发音→播完清除高亮 |
| 场景练习 | 预留功能入口 |

### 标签规范
- 只显示韩文（不显示罗马音）
- 韩文 + 音节拆解两行展示
- 标签放在空白区域，红色虚连线指向物品
- 点击后绿色边框高亮

### 图片生成prompt模板
```
A hand-drawn, cute cartoon style [场景名] interior.
Warm, cozy atmosphere. Soft colors. Cute flat illustration style.
No people. Clean composition. High quality, suitable for language learning.
[场景具体要求]
```

### 音频方案
- 工具：`edge-tts --voice ko-KR-SunHiNeural --text "소파" --write-media output.mp3`
- 集成：生成MP3 → base64编码 → 嵌入HTML → `new Audio('data:audio/mp3;base64,...')`
- ❌ 不用 Web Speech API（微信浏览器不支持）
- ❌ 不用 Google TTS（国内被墙）

### 场景规划（可扩展）

| 场景 | 优先级 | 状态 |
|:----|:-----:|:----:|
| 🛋️ 客厅 | P0 | ⏳ |
| 🛏️ 卧室 | P0 | ⏳ |
| 🍳 厨房 | P1 | ⏳ |
| ☕ 咖啡店 | P1 | ⏳ |
| 🏪 便利店 | P2 | 📝 |
| 🏫 学校/教室 | P2 | 📝 |
| 🏥 医院 | P2 | 📝 |

### 英语版扩展
- 只需替换语言数据（item的lang字段），模板完全复用
- 发音：`edge-tts --voice en-US-JennyNeural`
