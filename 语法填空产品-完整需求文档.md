# 语法填空智能练习产品 — 完整需求文档

---

## 一、产品定位

一款面向初中/高中学生的**语法填空在线练习工具**，基于你的英语教学体系，形成「做题 → 纠错 → 分析 → 精读 → 报告 → 针对性出题」的完整学习闭环。

老师课上可用，学生回家也能练。

---

## 二、目标用户

- **核心用户：** 你（英语老师），用于课堂教学 + 布置练习
- **最终用户：** 初中/高中学生
- **受众规模：** 🟡 中等

---

## 三、技术方案

| 项目 | 方案 |
|:--|:--|
| 前端 | HTML + CSS + JavaScript |
| 后端 | Node.js + Express（跑在你的腾讯云服务器上）|
| 数据存储 | SQLite（轻量数据库，无需额外安装）|
| AI出题 | 后端调中转API（灵眸AI）生成 |
| 学生数据 | 存服务器，换设备不丢 |
| 老师面板 | 查看全班学生的薄弱点统计 |

## 六、解题思路体系

基于你的教学思维导图 + 我的三点优化，每个空的解题思路按以下结构展示：

### 解题思路展示格式

```
📌 考点类型：有提示词 → 动词

① [决策树路径]
   ├─ 句子有其他谓语吗？→ 有/没有
   ├─ 主动还是被动？
   ├─ 什么时间？
   └─ 得出结论

② 易错提醒
   └─ 常见错误方向

③ 正确答案
   └─ 解析一句话总结
```

### 决策树一：有提示词 → 动词

```
看到动词提示词
       │
句子有其他谓语动词吗？
    ┌──┴──────────┐
    没有（缺谓语）     有（不缺谓语）
       │                │
   ↓ 判断时态        ↓ 用非谓语
   ├─ 时间词/语境      │
   ├─ 主谓一致        ┌──┴──────────────┐
   └─ 主动/被动    动作与主语的关系   动作与谓语的时间关系
                         │                 │
                     ┌───┴───┐        ┌───┴───┐
                   主动      被动      同时   已发生  未发生
                     │        │        │       │      │
                   doing    done     doing   done   to do
                              │                 │
                            还要将来?          还要被动?
                              │                 │
                            to be done      to be done
```

### 决策树二：有提示词 → 形容词/副词

```
看到形容词/副词提示词
       │
有比较信号词？（than/even/much/a lot/far）
    ┌──┴──────────┐
    有比较信号        没有比较信号
       │                │
   ↓ 判断比较级       ↓ 判断最高级
   比较级+than        ├─ the + 最高级
                      └─ of/in 范围
       │                │
   还要判断：形容词还是副词？
   ├─ 修饰名词/系动词后作表语 → 形容词
   └─ 修饰实义动词/形容词/全句 → 副词
```

### 决策树三：无提示词 → 从句引导词

```
看到空位 → 怀疑是从句
       │
   确定从句类型
    ┌──┴──────────────────────────┐
  定语从句               名词性从句              状语从句
     │                      │                      │
 看先行词               看缺不缺成分            看逻辑关系
 ├─ 人 → who/that      ├─ 缺"什么"→ what     ├─ 时→when/as
 ├─ 物 → which/that    ├─ 缺"是否"→ whether   ├─ 因→because
 ├─ 所属 → whose       └─ 不缺 → that         ├─ 条件→if/unless
 ├─ 地点 → where                              ├─ 让步→although
 └─ 时间→when                                 └─ 目的→so that
```

### 兜底策略一：遇到生词怎么办？

```
遇到生词，靠结构线索猜：
├─ 后缀判词性：-tion名词 / -ous形容词 / -ly副词
├─ 位置判成分：冠词后→名词 / 介词后→宾语
└─ 即使不认识，也知道该填什么词性/形式
```

### 兜底策略二：长难句三步剥洋葱

```
第一步：找谓语动词 → 确定有几句话
第二步：去掉修饰（介词短语/从句）→ 找到主谓宾
第三步：即使没全懂，也能通过结构填对
```

### 多解法优先级规则（非谓语）

```
非谓语有几种解法，按优先级走：
第一：看主动/被动（最可靠）
第二：看同时/之前/之后（缩小范围）
第三：看目的/意图（最后验证）
```

## 七、错因分类体系

| 错误类型 | 说明 |
|:--|:--|
| 时态/语态错误 | 时态判断错、主动被动混淆 |
| 非谓语形式错误 | doing/done/to do 选错 |
| 介词搭配错误 | 固定搭配不熟、介词误用 |
| 从句引导词错误 | which/that/what/when 混淆 |
| 词性判断错误 | 形容词该用副词、名词该用动词 |
| 长难句理解错误 | 句子结构没看懂导致填错 |
| 生词导致错误 | 不认识关键词 |
| 其他 | 粗心、拼写错误等 |

## 十二、部署方案

- 前端文件 + 后端代码都部署在你的**腾讯云轻量服务器**上
- 后端用 Node.js + Express，启动一个 Web 服务
- 学生通过 `http://你的服务器IP:端口` 访问
- 数据存 SQLite 文件，备份只需拷一个文件

## 十三、约束条件

- 需要你的腾讯云服务器跑后端（已有 Node.js）
- 学生需有简单账号（老师分配，无需手机号）
- AI出题通过中转API（灵眸AI）实现
- 第一版不做拍照和语音识别
- 解题思路严格基于你提供的教学体系
