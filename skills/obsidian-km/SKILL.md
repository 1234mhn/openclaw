# Obsidian 知识管理

## 核心概念

### 知识体系框架

| 库 | 子库 | 场景/分类 |
|---|---|---|
| 语言库 | 英语库 | 起床/通勤/点餐/职场/购物/旅行 |
| 语言库 | 韩语库 | 日常/点餐/问路/职场/购物 |
| 知识库 | AI相关 | 工具/理论/实操 |
| 知识库 | 基础知识 | 通用技能/学习方法 |
| 知识库 | 学科知识 | 课程笔记/考试准备 |

### 卡片标准字段（6字段）

每条笔记使用这 6 个字段：

```
① 📅 时间/日期
② 🏷️ 主题（如"英语-起床场景"）
③ 📖 大体内容（知识点摘要）
④ ❓ 遇到的问题/不理解的点
⑤ 🔧 解决过程（怎么搞懂的步骤）
⑥ 🚀 应用落地（怎么用起来）
```

### 仓库路径

```
OBSIDIAN_VAULT = /root/obsidian_vault
```

所有文件操作都以此路径为根。

## 核心流程

### 流程1：内容分类与卡片生成（通用入口）

收到用户输入的任何学习内容后：

1. **分类判断** — 输入内容属于哪个领域？
   - 英语/韩语词汇/短语/句子 → 走**语言库流程**
   - AI/技术/学科/方法论知识 → 走**知识库流程**
   - 复盘/复习请求 → 走**复盘流程**

2. **生成卡片文件** — 按 6 字段结构组织内容，使用 `📅YYYY-MM-DD-主题.md` 命名格式

3. **归档保存** — 写入对应目录，路径格式：
   - 语言：`{OBSIDIAN_VAULT}/语言库/XX语库/场景/📅YYYY-MM-DD-主题.md`
   - 知识：`{OBSIDIAN_VAULT}/知识库/大类/子类/📅YYYY-MM-DD-主题.md`

4. **更新索引** — 如果涉及新知识点，在对应的索引文件中追加链接

### 流程2：语言库管理（英语/韩语）

**触发条件**：用户输入词汇/短语/句子/语法点，或要求整理语言学习内容。

**处理步骤**：

1. **场景匹配** — 判断属于哪个场景（起床/通勤/点餐/职场/购物/旅行等）
2. **生成双语卡片**：
   - 核对每个词/短语的含义和用法
   - 提供中英/中韩对照
   - 标注使用场景说明
   - 如用户提供发音/录音，转录为文字并纠错
3. **存入对应场景目录**，命名：`📅YYYY-MM-DD-{场景}-{关键词}.md`
4. **不定期激活** — 从已学场景中抽取内容，以提问/造句/对话方式让用户主动回忆
   - 纠正错误后更新到原卡片

**目录路径**：
- 英语：`{OBSIDIAN_VAULT}/语言库/英语库/{场景}/`
- 韩语：`{OBSIDIAN_VAULT}/语言库/韩语库/{场景}/`

### 流程3：知识库管理（AI/基础/学科）

**触发条件**：用户输入知识点、技术概念、方法论、课程笔记等。

**处理步骤**：

1. **分类匹配** — 判断属于哪个大类+子类
2. **生成深度卡片**（6字段 + 额外深度扩展）：
   - 基础内容：完成6个标准字段
   - 深度扩展：
     - ① 识别并解释用户可能不熟悉的知识点
     - ② 挖掘延伸应用场景
     - ③ 关联其他知识点，连点成网
3. **存入对应目录**，命名：`📅YYYY-MM-DD-{主题}.md`
4. **不定期激活** — 抽取已存知识点，提问核心概念/应用方法/解决步骤

**目录路径**：
- AI：`{OBSIDIAN_VAULT}/知识库/AI相关/{工具|理论|实操}/`
- 基础：`{OBSIDIAN_VAULT}/知识库/基础知识/{通用技能|学习方法}/`
- 学科：`{OBSIDIAN_VAULT}/知识库/学科知识/{课程笔记|考试准备}/`

### 流程4：复盘与复习

**触发条件**：用户说"复盘""复习""回顾""测试我"，或复盘定时器触发。

**处理步骤**：

1. **随机抽取** 1-3 个知识点（语言库/知识库混合随机）
   - 从语言库英语/韩语各场景中抽取
   - 从知识库各子类中抽取
2. **以交互方式提问**：
   - **语言类**：给出中文让用户翻译/造句/对话模拟
   - **知识类**：提问核心概念、应用方法、解决步骤
3. **纠正与更新** — 用户回答后纠正错误，将复盘结果追加到原卡片末尾
4. **记录复盘** — 在复盘记录目录中保存本次复盘摘要：`{OBSIDIAN_VAULT}/复盘记录/📅YYYY-MM-DD-复盘.md`

## Obsidian 文件操作规范

### 文件命名
- 使用 `📅YYYY-MM-DD-{主题}.md` 格式
- 主题使用中文关键词，如 `📅2026-05-18-英语起床场景-go-get-ready.md`

### 卡片文件结构
```markdown
---
created: YYYY-MM-DD HH:mm
tags:
  - card
  - {lang|knowledge}
  - {场景/分类}
---

# {主题}

**📅 时间：** YYYY-MM-DD HH:mm
**🏷️ 主题：** {主题名称}
**📂 分类：** {库名}/{子库名}/{场景或子类}

---

## 📖 大体内容

{知识点完整摘要}


## ❓ 遇到的问题 / 不理解的点

{困惑和难点}


## 🔧 解决过程

{解决步骤}


## 🚀 应用落地

{实际应用方式}
```

### 索引文件更新
每次新增卡片后，检查对应目录的索引文件（如 `📋 英语索引.md`），若无链接则追加。

### Git 同步
vault 已配置 git 自动同步。创建/修改文件后无需手动 git 操作，sync.sh 会处理。
