# 知识卡片锻造系统

## 核心理念

一张好卡片不是记录，是**锻造**。把模糊的经验锤打成可行动的决策工具。

**四铁律：**
1. **原子性** — 一张卡只承载一个独立知识点
2. **可行动** — 读完能直接指导行为
3. **可证伪** — 必须能说清"这张卡何时不适用"（红灯不能为空）
4. **诚实性** — 绿灯场景必须来自真实使用经历

---

## 卡片结构

使用以下模板锻造新卡：

```
📇 卡片名称：[简短命名]
🔖 卡片ID：KC-[领域缩写]-[序号]
📊 等级：L1
💡 核心洞察（一句话）
[用最精炼的语言表达知识要点]

📐 行动指南
[具体怎么用、步骤或决策规则]

🟢 绿灯（适用场景）
· [场景1]
· [场景2]

🔴 红灯（不适用场景）
· [场景1]
· [场景2]

🔗 关联卡片
· [关系] KC-XXX：[说明]
```

### 等级定义

| 等级 | 含义 | 要求 |
|------|------|------|
| L1 初铸 | 通过四铁律 | ≥1绿灯 + ≥1红灯 |
| L2 试炼 | 在真实场景中使用过 | L1 + ≥2个真实使用记录 |
| L3 淬火 | 场景跨越维度差异 | L2 + 使用场景跨越≥2个维度 |

## 工作流程

### 1. 锻造新卡 `/锻造`

当用户说"锻造这张卡""把这段做成卡片"时：

1. **提取核心洞察** — 从用户输入中提炼一句话知识点
2. **填写卡片模板** — 按上方结构逐个字段填写
3. **检查四铁律** — 逐条验证，不合格则追问补充
4. **分配卡片ID** — `KC-[领域]-[序号]`，按已有卡片递增
5. **写入文件** — 保存到 `{OBSIDIAN_VAULT}/知识库/基础知识/读书笔记/` 目录
6. **更新索引** — 在索引文件中追加卡片链接

### 2. 迭代卡片 `/迭代 [卡片ID]`

当用户说"更新这张卡""加个新场景"时：

1. 读取原卡片文件
2. 按用户提供的新内容更新字段
3. 保持四铁律完整性
4. 回到原文件

### 3. 升级等级 `/升级 [卡片ID]`

当用户说"升级这张卡"时：

1. 读取卡片当前等级
2. 审查使用记录/场景证据
3. 判断是否满足升级条件（L1→L2需2个真实使用；L2→L3需跨维度场景）
4. 更新卡片元数据

### 4. 评估卡片 `/评估 [卡片ID]`

当用户说"体检这张卡""这卡好不好"时：

1. 逐条检查四铁律通过情况
2. 检查绿灯红灯是否真实不空洞
3. 给出改进建议
