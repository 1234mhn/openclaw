# 🎯 小红书笔记数据分析技能

## 2026 CES评分公式

```
CES = 点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8
```

| 互动 | 权重 | 策略意义 |
|:---|:---:|:---|
| 关注 | **×8** | 最高权重。长期粉丝价值(LTV)。内容要有关注钩子 |
| 评论 | **×4** | 社交传播力。要用提问式结尾/评论区引导 |
| 转发 | **×4** | 内容扩散力。干货/清单/合集类容易获得转发 |
| 点赞 | **×1** | 基础互动。操作成本最低，是数据基数 |
| 收藏 | **×1** | 内容价值感。藏播比>1%算有价值 |

## 2026流量池层级

| 层级 | 浏览范围 | 核心考核指标 | 关键动作 |
|:---:|:---:|:---|:---|
| 冷启动池 | **0~200** | 前2小时**点击率(CTR)** | 封面+标题决定生死 |
| 初级池 | **200~500** | **CES互动分** | 引导评论+收藏 |
| 中级池 | **500~5000** | **完播率+停留时长** | 前3秒钩子+内容节奏 |
| 热门池 | **1w+** | 人工审核+综合数据 | 内容合规+持续互动 |

## 2026算法新趋势

1. **搜索流量崛起（占约30%）** — 标题/正文前30字埋长尾关键词
2. **完播率 > 点击率** — 前3秒必须有钩子
3. **收藏=二次曝光** — 内容"值得存"比"值得看"更重要

## 分析流程

### Step 1 — 收集数据
用户提供：播放量、点赞、收藏、评论、转发、涨粉数、发布时间、笔记类型

### Step 2 — 计算核心指标

| 指标 | 公式 | 判断标准 |
|:---|:---|:---|
| 赞播比 | 点赞÷播放 | >3% ✅ >5% 🎯 |
| 藏播比 | 收藏÷播放 | >1% ✅ >3% 🎯 |
| 评播比 | 评论÷播放 | >0.5% ✅ >1% 🎯 |
| CES总分 | 各互动×权重求和 | 越高越好 |
| 涨粉转化 | 本期涨粉÷播放 | >1% ✅ |

### Step 3 — 诊断流量卡点

```
播放<200   → 封面/标题点击率不够（冷启动失败）
播放200~500 → CES互动分不足（缺评论/关注引导）
播放500~5000 → 完播率/停留时长不足（内容节奏问题）
播放>5000   → 人工审核/话题广度
```

### Step 4 — 数据优化建议

**要求：每条建议必须给出具体可执行的方案，不只是方向性提示。**

| 问题 | 建议（要详细到可直接执行） |
|:---|:---|
| 点击率低 | 给具体封面文案+标题方案，如「焦虑不是你的敌人｜3个方法马上缓解」 |
| 评论少 | 给出评论区置顶话术模板，如「说一个我的：每次上台汇报手心全是汗👇」 |
| 收藏低 | 给出收藏引导话术，如「先收藏⭐️下次焦虑时翻出来看」 |
| 关注少 | 给出下期预告话术，如「下期讲焦虑到失眠怎么办，关注不迷路」 |
| 完播低 | 给出前3秒完整脚本（逐字稿），如「第1秒说XX，第2~3秒说XX」 |

### Step 5 — 内容质量6维度分析

结合 `references/内容分析框架.md` 的6个维度，对内容进行定性分析。

**要求：每个维度必须给出具体判断依据，不只是打分。**

| 维度 | 核心问题 | 评分 | 分析要求 |
|:---|:---|:---:|:---|
| ①认知层 | 卖书还是卖改变？ | ✅/🟡/❌ | 说明内容是否打破固有认知，给出具体例子 |
| ②用户需求 | 满足什么需求？ | ✅/🟡/❌ | 结合平台趋势和评论洞察，给出判断依据 |
| ③内容结构 | 有递进逻辑吗？ | ✅/🟡/❌ | 分析结构层级，指出钩子设计的位置和效果 |
| ④行动导向 | 用户看完能做什么？ | ✅/🟡/❌ | 评估动作门槛，指出可优化的指令设计 |
| ⑤个人IP | 形象立住了吗？ | ✅/🟡/❌ | 分析人设真实感，给出增强代入感的具体建议 |
| ⑥变现逻辑 | 路径清晰吗？ | ✅/🟡/❌ | 分析变现方向，指出可持续优化的空间 |

### Step 6 — 输出完整复盘报告

**报告格式要求：**

```
📊 7月X日复盘报告 — [笔记标题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━ 📈 数据层面（完整）━━━━━━━━━━━
曝光/观看/点击率/完播率/2s退出率/涨粉
→ 逐一解读每个指标的含义和问题
→ 结合时间轴数据，指出表现最好的时段

━━━ 📋 6维度内容分析（逐维度）━━━━
每个维度：评分 + 判断依据 + 好在哪里/差在哪里

━━━ 💡 优化方案（详细可执行）━━━━━
P0 优先级最高—[问题]→[当前做法]→[改为做法]→[预期效果]
  要求：每一步都给出具体脚本/话术/方案
P1 ...（同上）
P2 ...（同上）

━━━ 🎯 优先级排序 ━━━━━━━━━━━━━━
按影响程度从高到低排列，标出紧急程度
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**⚠️ 每条优化建议必须包含：**
1. 当前存在的问题（数据或内容证据）
2. 具体的改进方案（不是方向，是完整方案）
3. 预期效果（改了之后会怎样）
4. 如果是视频内容，前3秒必须给出逐字稿
