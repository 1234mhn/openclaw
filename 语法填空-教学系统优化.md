## 优化一：决策树（先看什么，再看什么）

### 🌳 有提示词 — 动词

```
看到动词提示词
       │
       ▼
句子中还有其他谓语动词吗？
       │
    ┌──┴──────────┐
    没有（缺谓语）     有（不缺谓语）
       │                │
   ↓ 判断时态        ↓ 用非谓语
   ├─ 时间词/语境      │
   ├─ 主谓一致       ┌──┴──────────────┐
   └─ 主动/被动      动作与主语的关系   动作与谓语的时间关系
                    │                │
                ┌───┴───┐       ┌───┴───┐
              主动      被动     同时   已发生  未发生
                │        │       │       │      │
              doing    done    doing   done   to do
                        │                      │
                      还要将来?             还要被动?
                        │                      │
                      to be done            to be done
```

**学生口诀：** 一找谓语、二看关系、三定时间

---

### 🌳 有提示词 — 形容词/副词

```
看到形容词/副词提示词
       │
       ▼
有比较信号词吗？（than/even/much/a lot/far）
       │
    ┌──┴──────────┐
    有比较信号        没有比较信号
       │                │
   ↓ 判断比较级       ↓ 判断最高级
   比较级+than        ├─ the + 最高级
                      ├─ of/in 范围
                      └─ ever/never 语境
       │                │
       ▼                ▼
  还要判断：形容词还是副词？
    ├─ 修饰名词 / 系动词后作表语 → 形容词
    └─ 修饰实义动词 / 形容词 / 全句 → 副词
```

**学生口诀：** 先找信号、再判级、最后定词性

---

### 🌳 无提示词 — 从句引导词

```
看到空位 → 怀疑是从句
       │
       ▼
先确定是什么从句
       │
    ┌──┴──────────────────────────┐
    定语从句                      名词性从句              状语从句
    空在名词后                    空在动词前/后/句首      空连接两个完整的句子
       │                            │                      │
  ↓ 看先行词是人还是物          ↓ 看从句缺不缺成分        ↓ 看两句的逻辑关系
  ├─ 人 → who/whom/that       ├─ 缺"什么东西/人"       ├─ 时间 → when/while/as
  ├─ 物 → which/that              → what               ├─ 原因 → because/since
  ├─ 所属 → whose             ├─ 缺"是否"              ├─ 条件 → if/unless
  ├─ 地点 → where                → whether/if          ├─ 让步 → although/though
  ├─ 时间 → when              └─ 不缺成分              ├─ 目的 → so that
  └─ 原因 → why                  → that                └─ 结果 → so...that
                                                          │
                                                     【快速判断法】
                                                     ├─ "因为" → because
                                                     ├─ "如果" → if
                                                     ├─ "虽然" → although
                                                     └─ "当" → when/while
```

**学生口诀：** 先定从句类型、再判断引导词充当什么成分

## 优化二：生词 & 长难句兜底策略

### 🆘 遇到不认识单词怎么办？

```
遇到生词，别放弃！靠结构线索猜：
       │
    ┌──┴──────────────────────────┐
    判断这个词的词性               判断这个词的成分
       │                            │
  ├─ 后缀判断词性              ├─ 在动词后 → 可能是宾语
  │  -tion/-ment/-ness → 名词  ├─ 在介词后 → 一定是宾语
  │  -ous/-ful/-able → 形容词  ├─ 在冠词后 → 一定是名词
  │  -ly → 副词                └─ 在系动词后 → 可能是表语
  │  -ize/-ify → 动词
  │
  └─ 即使不认识，也能知道——
      该填名词还是形容词，该填什么形式
```

### 🆘 遇到长难句看不懂怎么办？

```
长难句 → 三步剥洋葱法：
       │
       ▼
第一步：找主干
  ├─ 找谓语动词（一个句子有几个谓语？）
  ├─ 去掉修饰（介词短语/从句/分词短语）
  └─ 剩下主谓宾核心
       │
       ▼
第二步：辨连接
  ├─ 连词是线索（and/or/but/because/if/when...）
  ├─ 从句引导词告诉你这是什么从句
  └─ 逗号往往是分界线
       │
       ▼
第三步：填答案
  ├─ 即使句子没全懂
  ├─ 通过结构+词性也能填对
  └─ 例：_____ (interesting) → 定语位置 → interesting
```

**长难句实例：**
```
The book (which was written by a famous author) ______ (sell) well.
                                    │
                            去掉定语从句修饰
                                    │
                           The book ______ well.
                                    │
                           缺谓语 → sells/sold
```

## 优化三：多解法优先级规则

### 🎯 非谓语多解法时，先走哪条路？

```
非谓语有几种解法 → 按优先级走：
       │
  第一优先级：看主动/被动（最可靠）
  ├─ 与逻辑主语是主动关系 → doing / to do
  └─ 与逻辑主语是被动关系 → done / to be done
       │
  第二优先级：看时间关系（缩小范围）
  ├─ 与谓语同时发生 → doing
  ├─ 发生在谓语之前 → done / having done
  └─ 发生在谓语之后 → to do
       │
  第三优先级：看目的/意图（最后验证）
  └─ 表目的 → to do
```

### 🎯 无提示词时，几个选项都能说得通怎么办？

```
比如：I'll call you _____ I get home.
如果学生想到 when / if / as / while 都行
       │
       ▼
优先级规则：
       │
  第一：看上下文确定逻辑关系
  ├─ 确定的时间 → when
  ├─ 不确定/条件 → if
  └─ 两个动作同时持续 → while
       │
  第二：看时态信号
  ├─ 主将从现 → 大概率是 when / if / as soon as
  ├─ 现在时 + 经常 → when / whenever
  └─ 完成时 → after / once / when
       │
  第三：验证语义最通顺的那个
```

## 三个优化一键总结

| 优化 | 解决什么问题 | 核心方法 |
|:--|:--|:--|
| ① 决策树 | 学生知道"有什么"，但不知道"怎么选" | 画清楚每一步先看什么、再看什么 |
| ② 兜底策略 | 遇到生词/长句直接放弃 | 不看懂也能靠结构填对 |
| ③ 多解法优先级 | 几种方法都能做，不知道该走哪条 | 排好优先级，先试最可能的那条 |

---

> 💡 下一步建议：你可以把这三个优化融入到你的思维导图里，每个分支旁边加上决策路径，而不是只列知识点。
