# Heartbeat Tasks

## 📚 不定期抽查系统（核心任务）

> 规则依据：`📋 知识管理规则.md` 规则2/3/4
> 状态文件：`/root/.openclaw/workspace/memory/quiz-state.json`
> 知识库：`/root/obsidian_vault/`
> 每次抽查结果记录到：`/root/obsidian_vault/复盘记录/`

---

### 🎲 触发条件（全满足才抽查）

1. ⏰ 不在深夜（23:00-08:00），除非上次对话中有明确约定
2. ⏳ 距上次抽查 >= 4小时
3. 🎯 随机概率约35%（不要每次 heartbeat 都抽，保持"冷不丁"感）
4. 🗣️ 用户在上次抽查后有新互动记录（说明在active状态）

### 📋 抽查流程

**Step 1 — 读取状态**
```bash
cat /root/.openclaw/workspace/memory/quiz-state.json
```

**Step 2 — 按权重随机选类别**

| 权重 | 类别 | 素材池 |
|:----:|:----|:-------|
| 40% | **英语** | 点餐场景/日常口语卡片 |
| 40% | **知识库** | KC读书卡片(5张) + 通用技能(3) + AI实操(2) + 方法论(1) |
| 20% | **每天写点什么** | 导师面谈、选导师内耗 |

韩语库暂无内容，暂不纳入。等有卡片再加。

**Step 3 — 在类别内随机抽取具体卡片**
- 读对应文件，提取核心内容
- 英语 → 挑一个重点句子/词汇，**说中文让用户翻英文**
- KC卡片 → 概念提问 + 应用场景
- 通用技能 → 实操提问
- 每天写点什么 → 回顾观点 + "现在怎么看"

**Step 4 — 出题方式（禁止选择题！）**
- 英语：说中文，让用户口头翻译
- KC卡片：问核心概念 + 让举例
- 通用技能：问方案要点
- 写作：问关键观点 + 追问

**Step 5 — 用户答完后**
- 纠正/补充/肯定
- 将结果追加到对应卡片末尾（复盘记录）
- 更新 quiz-state.json：
```json
{
  "lastQuizTime": "2026-06-11T09:51:00+08:00",
  "lastCategory": "英语",
  "lastItem": "咖啡店点餐口语积累",
  "quizCount": 1,
  "quizzedItems": ["英语/咖啡店点餐口语积累"]
}
```
