# Heartbeat Tasks

## 📚 不定期抽查系统（核心任务）

### 🤖 AI/ML 深度学习复盘（新增专项）

> **核心理念：** 不考记忆，考理解。每个知识点要经过「概括→追问→串联→应用」四轮打磨。

**触发条件：** 用户当天有AI学习记录时，下次对话优先触发

**流程：**

**Step 1 — 一句话概括**
> 请用**一句话**说清楚昨天/今天学的这个概念是什么
> *例：「线性回归就是找一条直线去拟合数据点」*

**Step 2 — 追问深化**
基于他的概括，追问1-2个问题把理解往深推：
> *例：「那这条直线怎么才算'最好'？用什么标准衡量？」*

**Step 3 — 串联旧知识**
把新学的内容跟之前学过的联系起来：
> *例：「这个跟你之前学的线性回归比，区别在哪？」*

**Step 4 — 应用检验**
让他想想这个知识点能用在UNet项目哪：
> *例：「这个损失函数能在你的CT分割项目上用吗？怎么用？」*

**如果答不上来：** 不要直接给答案，引导他想。实在想不起来再解释，解释完让他用自己的话再说一遍。

**记录方式：** 每次复盘结果追加到对应卡片末尾的「复盘记录」区块，标记第几轮打磨。

> 规则依据：`📋 知识管理规则.md` 规则2/3/4
> 状态文件：`/root/.openclaw/workspace/memory/quiz-state.json`
> 知识库：`/root/obsidian_nanan_daily_routine/`
> 每次抽查结果记录到：`/root/obsidian_nanan_daily_routine/复盘记录/`

### 🎲 触发条件（全满足才抽查）

1. ⏰ 不在深夜（23:00-08:00），除非上次对话中有明确约定
2. ⏳ 距上次抽查 >= 4小时
3. 🎯 随机概率约35%（不要每次 heartbeat 都抽，保持"冷不丁"感）
4. 🗣️ 用户在上次抽查后有新互动记录（说明在active状态）

### 📋 抽查流程

**Step 1 — 读取状态**
```bash
cat /root/.openclaw/workspace/memory/quiz-state.json
```

**Step 2 — 按权重随机选类别**

| 权重 | 类别 | 素材池 |
|:----:|:----|:-------|
| 30% | **英语** | 点餐场景/日常口语卡片 |
| 30% | **知识库** | KC读书卡片 + 通用技能 + 方法论 |
| 10% | **每天写点什么** | 导师面谈、选导师内耗 |
| **30%** | **🤖 AI/ML** | **新增！从AI相关卡片中抽取** |

韩语库暂无内容，暂不纳入。等有卡片再加。

**Step 3 — 在类别内随机抽取具体卡片**
- 读对应文件，提取核心内容
- 英语 → 挑一个重点句子/词汇，**说中文让用户翻英文**
- KC卡片 → 概念提问 + 应用场景
- 通用技能 → 实操提问
- 每天写点什么 → 回顾观点 + "现在怎么看"
- **🤖 AI/ML** → 从 `知识库/AI相关/理论/` 或 `知识库/AI相关/实操/` 中随机抽一张卡片

**Step 4 — 出题方式（禁止选择题！）**
- 英语：说中文，让用户口头翻译
- KC卡片：问核心概念 + 让举例
- 通用技能：问方案要点
- 写作：问关键观点 + 追问
- **🤖 AI/ML**：
  - 理论卡片 → 问核心概念 + 数学原理 + 跟其他模型的区别
  - 实操卡片 → 让口述代码逻辑 + 关键参数含义 + 跑代码时遇到的坑

**Step 5 — 用户答完后**
- 纠正/补充/肯定
- 将结果追加到对应卡片末尾（复盘记录）
- 更新 quiz-state.json：
```json
{

  "lastCategory": "英语",
  "lastItem": "咖啡店点餐口语积累",
  "quizCount": 1,
  "quizzedItems": ["英语/咖啡店点餐口语积累"]
}
```
