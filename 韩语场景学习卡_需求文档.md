# 韩语场景学习卡 — 需求文档（通用模板版）

## 一句话描述
一个可扩展的场景化韩语学习工具。**主页展示所有场景 → 选择一个场景 → 进入手绘卡通场景 → 点击物品学韩语**。每个场景共用同一套交互模板，只需替换场景数据和插画即可无限扩展。

---

## 技术栈
- **前端**：纯 HTML + CSS + JavaScript（单文件模式，后期可拆为多文件）
- **场景图**：由 **AI 绘图工具**（Midjourney / DALL-E / Stable Diffusion）生成手绘卡通风格 PNG/WebP，嵌入页面
- **发音**：用 **edge-tts**（微软韩语语音）预先生成 MP3 音频，**base64 内嵌**到 HTML 中，不依赖任何网络请求
- **数据驱动**：所有场景数据定义在 JS 对象中，新增场景只需加数据 + 加图片 + 加音频

---

## 目标用户 & 受众规模
- **核心用户**：自学韩语的中文母语者，初级水平
- **痛点**：坚持不下来 + 不知道每天学什么 + 缺乏沉浸式场景
- **规模**：先自用，再考虑分享/售卖（🟡 中等）

```javascript
        label: { left: "3%", top: "60%" },                               // 标签位置

```

### 交互模板逻辑（通用的，所有场景共用）

```
① 进入场景 → 
  显示该场景的 AI 手绘图片 + 所有物品的韩文标签（无罗马音）

② 点击物品 →
  · 点击区域高亮（绿色边框）
  · 对应标签高亮
  · 播放内嵌音频（韩语发音，edge-tts 生成）
  · 屏幕中央弹窗：显示韩文大字 + 中文释义

③ 全部朗读 →
  · 按顺序遍历所有物品，逐个高亮 + 发音

④ 场景练习 →
  · 弹出入口（后续功能预留）
```

### 连接线规则（通用）

每个物品的标签位置放在**场景空白区域**，用一条**红色虚连线 + 红点**指向实物：
- 线形：`stroke-dasharray="3,2"` 
- 颜色：`#c0392b`
- 透明度：0.45
- 端点：一个小红点 `r="2.5"`

## 场景图片生成规范（给 AI 绘图工具用）

### prompt 模板（通用）
```
A hand-drawn, cute cartoon style [场景名] interior.
Warm, cozy atmosphere. Soft colors. Cute flat illustration style.
No people. Clean composition. High quality, suitable for language learning.
[场景具体要求]
```

### 各场景 prompt 参考

| 场景 | 英文 prompt 关键词 |
|------|-------------------|
| 🛋️ 客厅 | living room with sofa, TV, coffee table, window, lamp, rug |
| 🛏️ 卧室 | bedroom with bed, closet, desk, lamp, window, mirror |
| 🍳 厨房 | kitchen with stove, refrigerator, sink, table, cabinets |
| ☕ 咖啡店 | cafe with counter, coffee machine, table, chairs, menu board |
| 🏪 便利店 | convenience store with shelves, fridge, counter, snacks |
| 🏫 学校 | classroom with desks, blackboard, chairs, window, bookshelf |

### 图片要求
- 格式：PNG / WebP
- 风格：手绘卡通（hand-drawn cartoon），非写实
- 构图：物品清晰可辨，有足够空白区域放标签
- 尺寸：建议 4:3 或 3:4 比例
- 无文字：图片本身不包含任何文字（文字由 HTML 层叠加）

## 音频方案（关键）

| 方法 | 说明 |
|------|------|
| ✅ **推荐：edge-tts CLI** | `edge-tts --voice ko-KR-SunHiNeural --text "소파" --write-media sofa.mp3` |
| 优点 | 微软语音，国内可用，音质好 |
| 集成方式 | 生成 MP3 → base64 编码 → 嵌入 HTML → `new Audio('data:audio/mp3;base64,...')` |
| 备选 | 若文件太大，可改为加载外部 MP3 文件（部署到服务器 / CDN） |

**注意：** 不要依赖 Web Speech API（微信浏览器不支持），不要用 Google TTS（国内被墙）

## 标签设计规范

**展示格式（无罗马音）：**
```
소파          ← 韩文：14px 加粗，深绿色
소·파         ← 韩语音节拆解：9px，棕色
```

- 每个物品两个标签都要展示，中间无间隔
- 标签背景：半透明白色卡片
- 边框：深红色
- 点击后高亮：绿色边框
- 所有标签放在场景空白区域，用连接线指向物品

## 交互流程

### 点击物品
```
用户看到标签 → 点击标签区域
    ↓
标签高亮（绿色边框）
↓
连接线高亮
↓
屏幕中央弹窗（韩文大字 + 中文）
↓
播放韩语发音（内嵌音频）
↓
1 秒后弹窗消失
```

### 全部朗读
```
点击「全部朗读」
    ↓
按顺序遍历所有物品
    ↓
逐个高亮 + 弹窗 + 发音
    ↓
播完后清除所有高亮
```

## 约束总结

- **场景图用 AI 生成**（Midjourney / DALL-E / SD），不要手写 SVG
- **音频用 edge-tts 生成后内嵌**，不要 Web Speech API 或 Google TTS
- **韩文标注只用韩文**，不要任何罗马音/英文注音
- **标签放在空白区域**，用连接线指向物品，不压在物品上
- **通用数据驱动模板**，新增场景只需加数据+图片+音频，不改 HTML 结构
- 手机优先，同时兼容桌面
